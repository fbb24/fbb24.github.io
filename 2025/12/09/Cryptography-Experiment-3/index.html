现代密码学实验三：RSA 公钥加密算法

课程名称 现代密码学 实验人 冯汝鑫
题目 实验三：RSA 算法实现与安全性分析 学号 23009200482

实验摘要

本实验围绕 RSA 公钥加密体制展开，分为两个部分。
首先，针对 Project Euler Problem 182，研究了 RSA 中的“不动点”（Unconcealed Messages）问题。通过分析数学原理，摒弃了低效的暴力遍历方法，采用基于 GCD 的公式快速计算特定 $p, q$ 下的不动点数量，从而筛选出最优的公钥指数 $e$。
其次，在 Cryptopals Set 5 的挑战中，从零实现了一个基础的 RSA 加密系统。实验中不依赖库函数，而是手动实现了扩展欧几里得算法以求解模逆元，并完成了素数生成、密钥对生成及加解密验证。
通过本次实验，深入理解了 RSA 的数学基石（欧拉定理、模逆运算）以及参数选择对安全性的影响。

一、 Project Euler 182: RSA Encryption

1.1 题目描述

题目来源：Project Euler Problem 182 。
在 RSA 系统中，对于给定的公钥 $(n, e)$，如果消息 $m$ 满足 $m^e \equiv m \pmod n$，则称 $m$ 为未被隐藏的消息（Unconcealed Message）或不动点。
题目给定 $p=1009, q=3643$，要求找出所有满足 $\gcd(e, \phi(n))=1$ 且 $1 < e < \phi(n)$ 的 $e$，使得未被隐藏的消息数量最少，并求这些 $e$ 的和。

1.2 理论分析

对于 RSA 加密 $c \equiv m^e \pmod n$，我们希望密文 $c$ 不等于明文 $m$。但在实际中，总存在一些 $m$ 使得加密无效（如 $m=0, m=1$）。

计算不动点数量的公式：
起初尝试遍历所有 $m$ 进行加密验证，但效率极低。根据数论推导，对于给定的 $n=pq$ 和 $e$，满足 $m^e \equiv m \pmod n$ 的消息 $m$ 的数量 $N$ 可以通过以下公式直接计算：

$$N = (1 + \gcd(e-1, p-1)) \times (1 + \gcd(e-1, q-1))$$

我们的目标是找到 $e$，使得 $N$ 最小。显然，$N$ 的最小值为 $p, q$ 互素情况下的极小值。

1.3 实验过程与代码

利用 gmpy2 库处理大数运算，根据上述公式筛选 $e$。

实验结果：
程序成功输出了使不动点最少的 $e$ 的列表及其总和。该方法避免了对 $m$ 的遍历，将复杂度降低到了仅与 $e$ 的搜索空间相关。

二、 Cryptopals Challenge 39: Implement RSA

2.1 题目描述

题目来源：Cryptopals Set 5 Challenge 39 。
要求实现一个简单的 RSA 算法。核心要求是不使用库函数求解模逆，而是手动实现扩展欧几里得算法（Extended Euclidean Algorithm）和模逆函数（Modular Inverse）。

2.2 算法原理

RSA 的安全性基于大整数分解的困难性。主要步骤如下：

  1. 密钥生成：
    • 生成两个大素数 $p, q$。
    • 计算 $n = p \times q$ 和 $\phi(n) = (p-1)(q-1)$。
    • 选择公钥指数 $e$，满足 $\gcd(e, \phi(n)) = 1$。
    • 计算私钥指数 $d$，使得 $ed \equiv 1 \pmod{\phi(n)}$。
  2. 加密：$c \equiv m^e \pmod n$。
  3. 解密：$m \equiv c^d \pmod n$。

扩展欧几里得算法 (Extended GCD):
用于求解 $ax + by = \gcd(a, b)$ 的整数解 $x, y$。当 $\gcd(a, b)=1$ 时，求得的 $x$ 即为 $a$ 模 $b$ 的逆元。

2.3 实验代码实现

三、 实验总结

本次实验涵盖了 RSA 算法的理论优化与基础实现：

  1. 关于不动点（Project Euler 182）：
    通过实验发现，RSA 算法并不总是能“隐藏”信息。在选择参数时，如果不刻意筛选，可能会存在较多的不动点。利用公式 $N=(1+\gcd(e-1,p-1))\times(1+\gcd(e-1,q-1))$ 可以快速评估 $e$ 的优劣。这提示我们在实际部署 RSA 时，除了素数的安全性，公钥指数的选择也需考究（虽然通常使用的 65537 已经是一个很好的权衡）。

  2. 关于 RSA 实现（Cryptopals 39）：
    手动实现 ex_gcd 让我对数论基础有了更直观的认识。然而，实验中的代码实现仅作为一个“教科书式”的原型（Textbook RSA），存在明显的安全隐患：

    • 参数过小：实验中为了演示仅使用了 64 位的素数，生成的 $n$ 极易被分解。实际应用至少需要 2048 位。
    • 公钥 $e$ 过小：实验中固定 $e=3$。虽然计算快，但在加密短消息或广播攻击场景下非常脆弱（例如 Coppersmith 攻击）。
    • 缺乏填充（Padding）：代码直接对转换后的整数进行运算。现实中必须使用 OAEP 或 PKCS#1 v1.5 填充方案，以引入随机性，防止对相同消息产生相同密文，并抵抗选择密文攻击。
